CramCraft

A website built for my class "Understanding Human Computer Interactions." I developed the entire front-end from raw HTML, CSS, and JavaScript, collaborating with a small group of people. We utilized GitHub for version control and Supabase to set up the database. Users can log in and create accounts, make their own flashcards for any subject, and share them. Unlike Quizlet, it features a unique option to print flashcards double-sided with terms and definitions flipped, allowing for easy cutting and physical use. It also includes a bulk import option for direct uploads from Quizlet.

Technologies Used:

  • HTML, CSS, JavaScript
  • GitHub (for collaboration)
  • Supabase (for database)

Smart Sorting Machine

This project showcases an optimized sorting algorithm that intelligently switches between QuickSort and MergeSort. I learned the fundamentals of both QuickSort and MergeSort in my Computer Science 1 (CS1) course, understanding their strengths and weaknesses. This implementation is designed to leverage QuickSort's typical speed while mitigating its worst-case performance by falling back to the consistently reliable MergeSort when an unbalanced partition is detected. Users can input numbers, load from a file, or generate a random list, then observe the sorting process and save the results.

Technologies Used:

  • HTML, CSS, JavaScript

Password Strength & Cracking Time Estimator

This tool provides instant feedback on password strength and estimates the theoretical time it would take for a brute-force attack to crack it. It analyzes the password's length and character composition (lowercase, uppercase, numbers, symbols) to calculate a strength score and estimate cracking time using a simplified model. Additionally, it offers a secure password generator that creates strong, random passwords ensuring a mix of character types. This project highlights an understanding of security principles and practical application of front-end development.

Technologies Used:

  • HTML, CSS, JavaScript

UCF Search

One of the earliest programs I developed independently. MyUCF, the university's portal, lacks a direct search feature. For example, to find "add classes," one typically has to navigate through multiple clicks: MyUCF -> Student Self Service -> Student Records -> Enrollment -> "Add Classes." This process is quite tedious. To streamline this, I created a program using HTML, CSS, and JavaScript where users can simply type what they're looking for and receive a direct link to the relevant MyUCF page, making navigation significantly easier.

Technologies Used:

  • HTML, CSS, JavaScript
